SonarLint for Visual Studio version 2.10-RC2

  • Release date: February 3, 2017
  • Release notes for SonarLint
  • Release notes for rules
  • Go to latest version

This version of SonarLint brings limited support for dataflow analysis on fields, thus directly improving several rules. We also fixed some bugs with the dataflow engine and have improved multiple rules. In addition, we improved the messages displayed when trying to connect to a SonaQube project whose quality profile contains no SonarAnalyzer rules (SonarLint does not support 3rd party rules). Here are the changes since version 2.9.0:

  • Enable dataflow analysis on all fields - SLVS-1129
  • Rule S1450 should not throw exceptions - SLVS-1142
  • Do not throw exceptions for new C# constructs in CFG and SE - SLVS-1143
  • Rule S1481 should not report on variables used inside lambdas - SLVS-1123
  • The symbolic value representing the result of nameof(xxx) should always have a NotNull constraint - SLVS-1124
  • Pointers should not be marked non-null by default - SLVS-1128
  • Connected mode should fail with an explicit message when there is no rules enabled for our analyzers - SVS-140
