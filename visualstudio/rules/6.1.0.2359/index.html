SonarLint for Visual Studio version 3.2

  • Release date: July 5, 2017
  • Release notes for SonarLint
  • Release notes for rules

This new version adds 6 new rules, fixes several bugs and brings improvements to symbolic execution-related rules. Here are the changes since version 3.1.0:

New Rules

  • Rule S2221: "Exception" should not be caught when not required by called methods
  • Rule S3927: Serialization event handlers should be implemented correctly
  • Rule S3972: Conditionals should start on new lines
  • Rule S4035: Classes implementing "IEquatable<T>" should be sealed
  • Rule S4039: Interface methods should be callable by derived types
  • Rule S4041: Type names should not match namespaces

Bug-fix and updates

  • Fix S1944: don't raise issue on "as"
  • Fix S2259: "Null pointer dereference" raises FP when a variable is initialized in catch(Exception) block
  • Fix S2386: Do not raise if the readonly field is initialized with a known immutable collection
  • Fix S2583: Rule should consider Nullable values
  • Fix S3897: Classes that provide "Equals(<T>)" should implement "IEquatable<T>"
  • Fix S3897: Correctly recognize IEquatable<T> implementations from base classes
  • Fix S4015: Rule should not raise on 'new' or 'override' members
  • Update S1479: "Switch with too many cases" should ignore empty, fall-through cases
  • Update S1764: Comparing the same object with Object.Equals() method
  • Update S3237: Add exception for empty properties from interfaces
  • Update S3427: "Method overloads" should not raise when methods differ by generic type arguments

Other improvements

  • Support nested try-catch blocks
  • Support many coverage reports
  • Potential problem in CFG with continue statement in and do-while loop
  • SonarC# fails when parsing xUnit test results with empty <assembly> tags
  • default() should have a null or not null constraint on it
  • Provide better error message when using old version of Scanner for MSBuild
  • Fix spelling errors
