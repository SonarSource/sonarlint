SonarLint for Visual Studio version 3.5

  • Release date: August 31, 2017
  • Release notes for SonarLint
  • Release notes for rules

This release adds 12 new rules, fixes 11 rules. Here are the changes since version 3.4.0:

New Rules

  • Rule S1168: Empty arrays and collections should be returned instead of null
  • Rule S1607: Tests should not be ignored
  • Rule S2183: Ints and longs should not be shifted by zero or more than their number of bits-1
  • Rule S2187: Test class should contain test methods
  • Rule S2699: Tests should include assertions
  • Rule S2701: Literal boolean values should not be used in assertions
  • Rule S3415: Assertion arguments should be passed in the correct order
  • Rule S3431: 'ExpectedExceptionAttribute' should not be used
  • Rule S3433: Test methods should have correct signature
  • Rule S4142: Duplicate values should not be passed as arguments
  • Rule S4144: Methods should not have identical implementations
  • Rule S4158: Empty collections should not be accessed or iterated

Bug fixes

  • Fix S1121: NullReferenceException when while loop with assignment expression is within a for loop with no condition
  • Fix S1751: Rule shouldn't raise on exceptions within while loops
  • Fix S2325: Should raise issues for methods and properties with SuppressMessage attribute
  • Fix S2372: Allow some exceptions being raised in properties as per CA1065
  • Fix S2372: do not throw ArgumentNullException on nlog
  • Fix S3242: Rule should not trigger on interface methods
  • Fix S3263: Rule should not raise when constant field is used in initialization
  • Fix S3877: do not throw ArgumentNullException on ravendb
  • Fix S3881: Rule should not fail with 'Syntax node is not within syntax tree'
  • Fix S3897: Rule should not suggest to implement IEquatable<T> when Equals is not public
  • Fix CognitiveComplexity to not throw an exception on Lucene.Net
