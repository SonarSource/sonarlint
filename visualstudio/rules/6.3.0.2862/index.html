SonarLint for Visual Studio version 3.4

  • Release date: August 2, 2017
  • Release notes for SonarLint
  • Release notes for rules

This release adds support for analyzing C/C++ code in Visual Studio, 4 new C# rules, fixes 8 C# rules. Here are the changes since version 3.3.0:

New Rules

  • Rule S1696: 'NullReferenceException' should not be caught
  • Rule S2114: Collections should not be passed as arguments to their own methods
  • Rule S3693: Exception constructors should not throw exceptions
  • Rule S3717: Track use of "NotImplementedException"

Bug fixes

  • Fix S100: infinite loop when class name contains non-Latin Characters
  • Fix S101: "Class Name" infinite loop when class name contains non-Latin Characters
  • Fix S1751: Rule should not raise on "retry on exception" pattern
  • Fix S2275: Rule should not trigger on space before alignment
  • Fix S3346: Rule raises FP on peach
  • Fix S3881: Rule should not fail on partial classes
  • Fix S3925: Call to 'base.GetObjectData' is not properly detected
  • Fix S3966: Rule should not throw cast exception

Others

  • Google.Protobuf.dll is not included in the dev vsix
  • POC for analyzer running on Linux
  • Symbolic Execution Engine supports multiple constraints per Symbolic Value
