This version of SonarLint brings support for JavaScript, 14 new rules and several fixes. Here are the changes since version 2.13.0:

New C# Rules

  • Rule S3981: Collection sizes and array lengths should not be tested against ">= 0"
  • Rule S3984: Exception should not be created without being thrown
  • Rule S3990: Assemblies should be marked as CLS compliant
  • Rule S3992: Assemblies should explicitly specify COM visibility
  • Rule S3993: Custom attributes should be marked with "System.AttributeUsageAttribute"
  • Rule S3994: URI Parameters should not be strings
  • Rule S3995: URI return values should not be strings
  • Rule S3996: URI properties should not be strings
  • Rule S3997: String URI overloads should call "System.Uri" overloads
  • Rule S3998: Threads should not lock on objects with weak identity
  • Rule S4000: Pointers to unmanaged memory should not be visible
  • Rule S4002: Disposable types should declare finalizer
  • Rule S4004: Collection properties should be readonly
  • Rule S4005: "System.Uri" argument should be passed instead of string

Fixes

  • Fix S1121: Assignments should not be made from within sub-expressions' should ignore chained assignments
  • Fix Sensors: Unit/Integration test results are skipped when module doesn't contain cs file
  • Fix S2275: InvalidCastExteption when analyzing single argument method

Updates

  • Update S3776: Disable by default because it is a parametrized rule
  • Update S2589 and S2583: raise issues for "true" and "false" literals
  • Update S3449: replace XML-encoded characters in description
  • Update S3881: should require a call to base.Dispose(bool)
  • Update S2372: "Do not throw exceptions from property getters" should be Code Smell
  • Update S112: "Do not throw general exceptions" should be Code Smell
  • Update S1854: Dead Stores should be code smell
  • Deprecate S1145
