SonarLint for Visual Studio version 3.3

  • Release date: July 19, 2017
  • Release notes for SonarLint
  • Release notes for rules

This version adds 20 new rules, 3 bug fixes and 2 rule updates. Here are the changes since version 3.2.1:

New Rules

  • Rule S1075: URIs should not be hardcoded
  • Rule S1200: Classes should not be coupled to too many other classes (Single Responsibility Principle)
  • Rule S3242: Method parameters should be declared with base types
  • Rule S3649: User-provided values should be sanitized before use in SQL statements
  • Rule S3900: Arguments of public methods should be validated against null
  • Rule S3956: "Generic.List" instances should not be part of public APIs
  • Rule S3966: Objects should not be disposed more than once
  • Rule S4047: Generics should be used when appropriate
  • Rule S4049: Properties should be preferred
  • Rule S4050: Operators should be overloaded consistently
  • Rule S4052: Types should not extend outdated base types
  • Rule S4055: Literals should not be passed as localized parameters
  • Rule S4056: Overloads with a "CultureInfo" or an "IFormatProvider" parameter should be used
  • Rule S4057: Locales should be set for data types
  • Rule S4058: Overloads with a "StringComparison" parameter should be used
  • Rule S4059: Property names should not match get methods
  • Rule S4060: Non-abstract attributes should be sealed
  • Rule S4061: "params" should be use instead of "varargs"
  • Rule S4069: Operator overloads should have named alternatives
  • Rule S4070: Non-flags enums should not be marked with "FlagsAttribute"

Bug-fix and updates

  • Fix S2386: Rule should not report when readonly field is initialized with null
  • Fix S3881: "Implement IDisposable correctly" should allow calling GC.SuppressFinalize(this) even when there is no destructor
  • Fix S3908: Rule should not report issues on classes that implement interfaces
  • Update S110: documentation should describe which delimiter to use or provide specific examples
  • Update S927: C#: parameter names should match base declaration and other partial definitions
