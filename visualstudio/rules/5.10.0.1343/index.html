This version of SonarLint brings a 30 new rules and several fixes. Here are the changes since version 2.12.0:

New Rules on SonarWay profile

  • Rule S2365: Properties should not make collection or array copies
  • Rule S1751: Jump statements should not be used unconditionally
  • Rule S2589: Boolean expressions should not be gratuitous
  • Rule S3915: Conditions should not unconditionally evaluate to "false"
  • Rule S3962: "static readonly" constants should be "const" instead
  • Rule S3971: "GC.SuppressFinalize" should not be called
  • Rule S3925: "ISerializable" should be implemented correctly
  • Rule S2365: Properties should not make collection or array copies
  • Rule S3928: Check ArgumentException instantiations for argument names

New Rules (disabled by default)

  • Rule S3967: Multidimensional arrays should not be used
  • Rule S3963: "static" fields should be initialized inline
  • Update S2070: the rule should include a longer list of deprecated hash algorithms
  • Update S2278: The rule should include a longer list of deprecated encryption algorithms

Bug-fix/F-P/Kill the Noise

  • Update S1144: Rule should handle unused internal types
  • Fix S2275: Rule should not report format string error on certain method when no parameters are provided
  • Fix S1698: Add exception for null comparison
  • Rule S2757 should not raise an issue when no whitespace after the two operators
  • Fix S1450 detects false positive with fields.
  • S1944 detects false Positive if casted to nullable
  • Update S110: Rule should add a filtering parameter to ignore some base classes
  • Fix S2275: Rule should not report on multiple ":" in formatting methods
  • Fix S3247: Rule should not report on is + cast for ValueTypes
  • Rule S2360 (Optional parameters) update to ignore non-public methods
  • Fix 1450: False positive in VS2017 but not VS2015
  • Fix S3236: Rule should no longer report on CallerMemberName
  • Improve S1144 (Unused private methods) messages to better suggest what's expected to be removed.
  • Rule S2372: it's fine to throw some types of exception from a getter
  • Fix S1451: Rule should handle multiple single-line comments
