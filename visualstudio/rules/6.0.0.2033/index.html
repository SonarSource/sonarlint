SonarLint for Visual Studio version 3.1 and 3.1.1

  • Release notes for SonarLint 3.1.1

SonarLint 3.1.1 fixes a connectivity problem with SonarCloud.io

This new version drops support for MSBuild 12, adds 15 new rules, adjusts the severity of some rules and fixes several bugs. Here are the changes since version 3.0.0:

New Rules

  • Rule S4040: Strings should be normalized to uppercase
  • Rule S4019: Base class methods should not be hidden
  • Rule S4015: Inherited member visibility should not be decreased
  • Rule S3908: Generic event handlers should be used
  • Rule S3906: Declare event handlers correctly
  • Rule S4027: Exceptions should provide standard constructors
  • Rule S4026: Assemblies should be marked with NeutralResourcesLanguageAttribute
  • Rule S3909: Collections should implement the generic interface
  • Rule S4018: Generic methods should provide type parameters
  • Rule S4022: Enums storage should be Int32
  • Rule S1123: "Obsolete" attributes should include explanations
  • Rule S4023: Interfaces should not be empty
  • Rule S2068: Credentials should not be hard-coded
  • Rule S4017: Method signatures should not contain nested generic types
  • Rule S4016: Enumeration members should not be named "Reserved"

Changes

  • Drop MSBuild 12 support Area
  • Process coverage reports only once for a solution
  • Decrease plugin size
  • Fix S1751: "Unconditional jump statements" should not raise if a conditional continue is detected
  • Fix S1940: "Inverted boolean checks" should not suggest inversion for Nullable
  • Fix Symbolic Execution: Invalid nameof() should be properly handled
  • Fix S3963: Should not raise issue when the static constructor does not assign any fields
  • Fix S2589: "Gratuitous expressions" should not raise if expression is the condition of a loop and break is detected
  • Fix S1871: "Branches with same implementation" should ignore single line blocks
  • Fix S1939: Extends and implements list entries should not be redundant
  • Update S2743: "Static fields in generic types" should be code smell
  • Update S3925: "ISerializable should be implemented correctly" should be code smell
  • Update S3220: "Method calls should not resolve ambiguously" should be code smell
  • Update S3881: "IDisposable should be implemented correctly" should be code smell
  • Update S3904: "Assemblies should have version information" should be code smell
  • Update S3885: "Assembly.Load should be used" should be code smell
  • Update S1210: "Override Equals when implementing IComparable" should be code smell
  • Update S2387: Rule S4025: "Child class fields shadowing parent fields" should ignore "static" fields, field names that differ by case are handled by S4025
  • Update S2234: Add support for constructors Type: False-Negative
  • Update S2328: "GetHashCode should not reference mutable fields" should report once per method
  • Update S2681: Fix FP on try-catch block Area: Rules Type: False-Positive
  • Update S3904: "Assemblies should have version" should be code smell
